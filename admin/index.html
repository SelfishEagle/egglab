Setup Guide

How to Test Locally (No npm, no terminal!)

Open the egglab folder in VS Code. Right-click index.html and choose Open with Live Server. Everything works locally — shop, cart, admin panel, order tracking, and even the full checkout flow. PayPal.me is just a normal link so it works everywhere.

Don't have Live Server? Install it from VS Code Extensions (search Live Server by Ritwick Dey). Or run python -m http.server 8080 in the folder if you have Python installed.

Step 1 — Fill in js/config.js

Open js/config.js and paste in your:

  • Supabase URL and anon key (from Supabase → Project Settings → API)
  • PayPal.me username — just the username part e.g. EggLabCreations

Step 2 — Create Supabase Tables

In Supabase → SQL Editor, paste and run this:

-- Products create table products ( id uuid default gen_random_uuid() primary key, name text not null, description text, price decimal(10,2) not null, stock integer default 0, category text, image_url text, created_at timestamptz default now() ); -- Orders create table orders ( id text primary key, customer_name text not null, customer_email text not null, customer_address text, total decimal(10,2) not null, status text default 'Processing', stripe_payment_intent text, created_at timestamptz default now() ); -- Order items create table order_items ( id uuid default gen_random_uuid() primary key, order_id text references orders(id), product_id uuid, product_name text not null, price decimal(10,2) not null, qty integer not null ); -- Row Level Security alter table products enable row level security; alter table orders enable row level security; alter table order_items enable row level security; -- Products: public read, authenticated write create policy "Public read products" on products for select using (true); create policy "Admin insert products" on products for insert with check (auth.role() = 'authenticated'); create policy "Admin update products" on products for update using (auth.role() = 'authenticated'); create policy "Admin delete products" on products for delete using (auth.role() = 'authenticated'); -- Orders: public insert/read, authenticated update create policy "Public insert orders" on orders for insert with check (true); create policy "Public read orders" on orders for select using (true); create policy "Admin update orders" on orders for update using (auth.role() = 'authenticated'); -- Order items: public insert/read create policy "Public insert order_items" on order_items for insert with check (true); create policy "Public read order_items" on order_items for select using (true);

Step 3 — Create Storage Bucket

In Supabase → Storage, create a bucket called exactly product-images and set it to Public. This lets you upload product photos from the admin panel.

Step 4 — Create Your Admin Account

In Supabase → Authentication → Users, click Invite user, enter your email, and follow the link in the invite email to set a password. Use that email and password to log into this admin panel.

Step 5 — Deploy to Netlify

Zip the entire egglab folder and drag-and-drop it on netlify.com → Sites → Deploy manually. That's it — no environment variables needed, no build steps, no npm. It's a fully static site now.

Set a custom domain in Netlify → Domain settings once it's live.

How Payments Work

When a customer checks out:

  • Their order details are saved to Supabase with status Pending Payment
  • They're shown their unique Order ID and a Pay via PayPal button
  • The button opens your PayPal.me page with the correct amount pre-filled
  • They're asked to include their Order ID in the PayPal payment note
  • Once you receive the PayPal payment, update the order status here to Processing → Shipped
